### 📊 Architectural Patterns

| Pattern | Description | Benefits |
|---------|-------------|----------|
| **Microservices** | Independent, decoupled services | Independent scaling & deployment |
| **API Gateway** | Single entry point | Centralized routing & authentication |
| **Service Discovery** | Automatic service detection (Consul/etcd, Docker DNS) | Dynamic configuration, HA |
| **Circuit Breaker** | Cascade failure prevention (Resilience4j, Polly) | Graceful degradation |
| **Event Sourcing** | Event storage (Redis Streams, Kafka) | Full audit trail, event replay |
| **CQRS** | Read/write separation (PostgreSQL + Redis) | Query optimization, scalability |
| **Saga Pattern** | Distributed transactions | Eventual consistency |
| **Rate Limiting** | Request throttling (Token bucket, Sliding window) | Abuse protection, stability |
| **Health Check** | Health monitoring (`/health` endpoints) | Early problem detection |

### 🔄 Inter-Service Communication

#### Synchronous (REST/HTTP)

```python
import httpx

async def call_service(service_url: str, endpoint: str, data: dict):
    async with httpx.AsyncClient(timeout=30.0) as client:
        response = await client.post(
            f"{service_url}/{endpoint}",
            json=data,
            headers={"Authorization": f"Bearer {token}"}
        )
        return response.json()
```

#### Asynchronous (Event-Driven)

```python
import redis.asyncio as redis

async def publish_event(stream: str, event: dict):
    r = await redis.from_url("redis://localhost:6379")
    await r.xadd(stream, event)

async def consume_events(stream: str, consumer_group: str):
    r = await redis.from_url("redis://localhost:6379")
    while True:
        messages = await r.xreadgroup(
            consumer_group, "consumer-1", {stream: ">"}, count=10
        )
        for stream, messages_list in messages:
            for msg_id, data in messages_list:
                await process_event(data)
```

```bash
# 1. Clone the repository
git clone <repository-url>
cd blatam-academy/agents/backend/onyx/server/features

# 2. Configure environment variables

# Edit .env with your settings

# 3. Start the full system

# 4. Check status
```

```bash
# Build and run all services
docker-compose -f docker-compose-all.yml up -d --build

# Watch logs in real time
docker-compose -f docker-compose-all.yml logs -f

# Check service status
docker-compose -f docker-compose-all.yml ps
```

#### Option 2: Manual Setup

```bash
# 1. Create virtual environment
python -m venv venv
source venv/bin/activate  # Windows: venv\Scripts\activate

# 2. Install dependencies
pip install -r requirements.txt

# 3. Set up the database
python scripts/setup_database.py

# 4. Start individual services
python start_system.py start --service integration_system
python start_system.py start --service bulk
```

#### Option 3: Per-Service Installation

```bash
# Install and run a specific service
cd music_analyzer_ai
pip install -r requirements.txt
python main.py

# Or with Docker
docker build -t music-analyzer .
docker run -p 8010:8000 music-analyzer
```

### ➕ Adding a New Service

1. **Create the service structure**
   ```bash
   mkdir new_service && cd new_service

   touch main.py requirements.txt README.md
   ```

2. **Implement the FastAPI app** — see [templates](docs/)

3. **Add to Docker Compose** — `docker-compose-all.yml`

4. **Register in Integration System** — `integration_system/config/services.py`

5. **Update Nginx** — `nginx/nginx.conf`

### Common Issues

| Problem | Quick Fix |
|---------|-----------|
| Service won't start | `docker-compose logs [service] --tail=100` |
| Port already in use | `netstat -ano | findstr :8000` (Windows) |
| Database connection error | `docker-compose restart postgres` |
| Out of memory | `docker system prune -a && docker-compose restart` |
| GPU not detected | Verify `nvidia-smi` and CUDA drivers |
| Network errors | `docker network inspect blatam-network` |

## 🤝 Contributing

Contributions are welcome! This project is community-maintained and we appreciate every contribution.

> 📖 **Full Guide**: See our [Contributing Guide](CONTRIBUTING.md) for complete details.

### How to Contribute

1. **Fork** the repository on GitHub
2. **Clone** your fork and create a feature branch
   ```bash
   git clone https://github.com/your-user/blatam-academy.git
   cd blatam-academy/agents/backend/onyx/server/features
   git checkout -b feature/my-new-feature
   ```
3. **Make changes** following project conventions (Black formatting, type hints, tests)
4. **Verify** your changes
   ```bash
   black . && flake8 . && mypy . && pytest
   ```
5. **Commit** with [Conventional Commits](https://www.conventionalcommits.org/)
   ```bash
   git commit -m "feat(api): add document analysis endpoint"
   ```
6. **Push** and open a Pull Request

### Contribution Types

| Type | Prefix | Example |
|------|--------|---------|
| 🐛 Bug Fix | `fix:` | `fix: resolve authentication error` |
| ✨ Feature | `feat:` | `feat: add analysis endpoint` |
| 📚 Docs | `docs:` | `docs: update installation guide` |
| ⚡ Performance | `perf:` | `perf: optimize database queries` |
| ♻️ Refactor | `refactor:` | `refactor: reorganize module structure` |
