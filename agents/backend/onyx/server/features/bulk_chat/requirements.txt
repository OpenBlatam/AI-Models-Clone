# INSTALACIÓN RÁPIDA - Mejores prácticas 2024-2025

# 1. USAR UV (más rápido que pip): pip install uv && uv pip install -r requirements.txt
# 2. Instalar solo core: uv pip install fastapi uvicorn pydantic
# 3. Para producción: usar versiones exactas (==) en lugar de >=
# 4. Algunas librerías requieren compilación (numba, numpy, etc.)
# 5. Para mejor rendimiento: uv pip install --no-cache-dir -r requirements.txt
# 6. Para desarrollo: usar uv venv para crear entorno virtual rápido
# 7. Compatibilidad: Python 3.10+ (recomendado 3.12+)

# ============================================================================
# ALTERNATIVAS RÁPIDAS - Librerías más rápidas disponibles
# ============================================================================
# - orjson: JSON más rápido que json estándar (C++)
# - polars: DataFrame más rápido que pandas (Rust)
# - duckdb: SQL más rápido que SQLite (C++)
# - simdjson: JSON parsing más rápido (SIMD)
# - hiredis: Redis parser más rápido (C)
# - py-spy: Profiler más rápido (Rust)

# ============================================================================
# NOTAS DE RENDIMIENTO
# ============================================================================
# - orjson es ~2-3x más rápido que json estándar
# - polars es ~5-10x más rápido que pandas para operaciones grandes
# - duckdb es ~100x más rápido que pandas para análisis OLAP
# - simdjson es ~2-3x más rápido que orjson para parsing puro
# - hiredis mejora Redis hasta 2x en throughput
# - lxml es ~10x más rápido que BeautifulSoup para parsing XML/HTML
# - websockets es más eficiente que python-socketio para conexiones puras
# - aiosmtplib es async y más eficiente que smtplib estándar

# ============================================================================
# RECOMENDACIONES POR USO
# ============================================================================
# Para JSON rápido: orjson > simdjson > rapidjson > json estándar
# Para DataFrames: polars > duckdb > pandas (según caso de uso)
# Para WebSockets: websockets (async) > python-socketio (compatibilidad)
# Para Scheduling: apscheduler (flexible) > croniter (simple)
# Para Testing: hypothesis (property-based) + pytest (unit)
# Para Templates: jinja2 (FastAPI usa esto) > mako
# Para Email: aiosmtplib (async) > smtplib estándar
# Para Profiling: py-spy (Rust, rápido) > pyinstrument > line-profiler
# Para Type Checking: mypy (completo) > pyright (rápido) > pyre-check
# Para Linting: ruff (Rust, ultra-rápido) > flake8 > pylint
# Para Code Formatting: black (estándar) > autopep8
# Para Task Queues: celery (completo) > dramatiq (simple) > arq (async puro)
# Para ORM: SQLAlchemy (completo) > SQLModel (Pydantic+SQLAlchemy) > Tortoise (async)
# Para Config: dynaconf (dinámico) > omegaconf (estructurado) > python-decouple
# Para Observability: OpenTelemetry (estándar) > Datadog > New Relic > Jaeger
# Para Debugging: debugpy (VS Code) > pudb (visual) > ipdb (interactivo)
