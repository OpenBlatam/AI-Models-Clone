## 📋 Requisitos

- Node.js 18+ 
- npm o yarn
- Backend BUL API corriendo (por defecto en `http://localhost:8000`)

## 🛠️ Instalación

1. **Instalar dependencias:**

```bash
npm install
# o
yarn install
```

2. **Configurar variables de entorno:**

Crea un archivo `.env.local`:

```env
NEXT_PUBLIC_API_URL=http://localhost:8000
```

3. **Ejecutar en desarrollo:**

```bash
npm run dev
# o
yarn dev
```

4. **Abrir en el navegador:**

```
http://localhost:3000
```

## 📦 Scripts Disponibles

- `npm run dev` - Inicia el servidor de desarrollo

- `npm run start` - Inicia el servidor de producción

## 🔌 Integración con Backend

El frontend se conecta automáticamente al backend BUL API usando:

- **REST API** para operaciones CRUD
- **WebSocket** para actualizaciones en tiempo real
- **Health checks** periódicos

### Endpoints Utilizados

- `GET /api/health` - Estado del sistema
- `GET /api/stats` - Estadísticas
- `POST /api/documents/generate` - Generar documento
- `GET /api/tasks` - Listar tareas
- `GET /api/tasks/{id}/status` - Estado de tarea
- `GET /api/tasks/{id}/document` - Obtener documento
- `DELETE /api/tasks/{id}` - Eliminar tarea
- `POST /api/tasks/{id}/cancel` - Cancelar tarea
- `GET /api/documents` - Listar documentos
- `WS /api/ws/{task_id}` - WebSocket para actualizaciones

## 🐛 Troubleshooting

**Error de conexión al backend:**
- Verifica que el backend esté corriendo en el puerto configurado
- Revisa la variable `NEXT_PUBLIC_API_URL` en `.env.local`

**Error de tipos TypeScript:**
- Ejecuta `npm run type-check` para ver errores
- Asegúrate de tener todas las dependencias instaladas

**WebSocket no funciona:**
- Verifica que el backend soporte WebSocket
- Revisa la consola del navegador para errores
