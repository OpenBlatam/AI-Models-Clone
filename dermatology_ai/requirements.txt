# ============================================================================
# Production Notes & Best Practices
# ============================================================================
# 1. Version Management:
#    - All versions are minimum versions (>=) for flexibility
#    - Use `pip-compile` or `poetry` for exact version pinning in production
#    - Consider using `pip-tools` to generate requirements-lock.txt
#    - Use `make compile` or `.pip-requirements-update.sh compile`
#
# 2. Installation Options:
#    - Production:        pip install -r requirements.txt
#    - Optimized:         pip install -r requirements-optimized.txt (~500 MB)
#    - Development:       pip install -r requirements-dev.txt
#    - Minimal:           pip install -r requirements-minimal.txt (~50 MB)
#    - GPU Support:       pip install -r requirements-gpu.txt
#    - Docker:            pip install -r requirements-docker.txt
#    - Or use Makefile:   make install / make install-opt / etc.
#
# 3. Serverless Deployment:
#    - Use `mangum` adapter for AWS Lambda
#    - Optimize with multi-stage Docker builds for containers
#    - Consider using Lambda Layers for large dependencies
#    - Use requirements-optimized.txt for smaller package size
#
# 4. Performance:
#    - Use `uvloop` on Linux/macOS for better async performance
#    - Use `orjson` for fastest JSON serialization
#    - Enable Redis caching for production
#    - Use ONNX Runtime for optimized ML inference
#
# 5. Security:
#    - Regularly update: make outdated or pip list --outdated
#    - Check vulnerabilities: make check or safety check
#    - Use `safety` or `pip-audit` to check for vulnerabilities
#    - Pin critical security packages to specific versions
#    - Review security advisories regularly
#
# 6. Monitoring:
#    - Enable OpenTelemetry for distributed tracing
#    - Use Prometheus metrics for monitoring
#    - Set up structured logging with structlog
#    - Monitor dependency updates with Dependabot/Renovate
#
# 7. Database:
#    - Use asyncpg for PostgreSQL (fastest)
#    - Use connection pooling for production
#    - Consider read replicas for read-heavy workloads
#    - Use Redis for caching frequently accessed data
#
# 8. ML/AI:
#    - Use ONNX for optimized inference (smaller, faster)
#    - Consider quantization for smaller models
#    - Use GPU runtime for production inference (requirements-gpu.txt)
#    - Use requirements-optimized.txt if only inference needed
#
# 9. Docker:
#    - Use requirements-docker.txt for containerized deployments
#    - Use multi-stage builds to reduce image size
#    - Use opencv-python-headless (no GUI dependencies)
#    - See Dockerfile examples in DEPENDENCIES_GUIDE.md
#
# 10. Development:
#     - Use `make` commands for common tasks (see Makefile)
#     - Use pre-commit hooks: make pre-commit
#     - Run tests: make test or make test-cov
#     - Format code: make format
#     - Type check: make type-check
#
