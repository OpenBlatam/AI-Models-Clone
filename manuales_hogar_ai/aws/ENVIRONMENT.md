# Environment Configuration Guide

## Required Environment Variables

### Application Level

These are set automatically by the ECS task definition:

- `PORT`: Application port (default: 8000)
- `DB_HOST`: RDS endpoint (automatically set)
- `DB_PORT`: Database port (default: 5432)
- `DB_NAME`: Database name (default: manuales_hogar)

### Secrets (Managed by AWS Secrets Manager)

These are retrieved from AWS Secrets Manager:

- `OPENROUTER_API_KEY`: Retrieved from `manuales-hogar-ai/openrouter-api-key`
- `DB_USER`: Retrieved from `manuales-hogar-ai/db-credentials` (username field)
- `DB_PASSWORD`: Retrieved from `manuales-hogar-ai/db-credentials` (password field)

### Optional Environment Variables

- `ENVIRONMENT`: Environment name (dev/staging/prod) - affects logging and reload behavior
- `WORKERS`: Number of Uvicorn workers (default: 4, only used in production)
- `LOG_LEVEL`: Logging level (default: INFO)

## Setting Secrets in AWS Secrets Manager

### OpenRouter API Key

```bash
aws secretsmanager create-secret \
    --name manuales-hogar-ai/openrouter-api-key \
    --secret-string '{"api_key": "your-api-key-here"}' \
    --region us-east-1
```

Or update existing:

```bash
aws secretsmanager update-secret \
    --secret-id manuales-hogar-ai/openrouter-api-key \
    --secret-string '{"api_key": "your-new-api-key"}' \
    --region us-east-1
```

### Database Credentials

Database credentials are automatically generated by RDS and stored in Secrets Manager. To update:

```bash
# Get current secret
aws secretsmanager get-secret-value \
    --secret-id manuales-hogar-ai/db-credentials \
    --region us-east-1

# Update password (you'll need to update RDS password separately)
aws rds modify-db-instance \
    --db-instance-identifier manuales-hogar-ai-database \
    --master-user-password "new-password" \
    --region us-east-1
```

## Environment-Specific Configurations

### Development

- `ENVIRONMENT=dev`
- Reload enabled
- Single worker
- Verbose logging

### Staging

- `ENVIRONMENT=staging`
- Reload disabled
- Multiple workers
- Standard logging

### Production

- `ENVIRONMENT=prod`
- Reload disabled
- Multiple workers (4+)
- Optimized logging
- Enhanced monitoring

## Updating Environment Variables

### Via CDK

Edit `aws/infrastructure/app.py` and redeploy:

```bash
cd aws/infrastructure
cdk deploy
```

### Via AWS Console

1. Go to ECS Console
2. Select cluster: `manuales-hogar-ai-cluster`
3. Select service: `manuales-hogar-ai-service`
4. Click "Update"
5. Modify task definition
6. Update environment variables
7. Save and deploy

### Via AWS CLI

```bash
# Get current task definition
aws ecs describe-task-definition \
    --task-definition manuales-hogar-ai-task-def \
    --region us-east-1 > task-def.json

# Edit task-def.json to add/modify environment variables

# Register new task definition
aws ecs register-task-definition \
    --cli-input-json file://task-def.json \
    --region us-east-1

# Update service
aws ecs update-service \
    --cluster manuales-hogar-ai-cluster \
    --service manuales-hogar-ai-service \
    --task-definition manuales-hogar-ai-task-def \
    --force-new-deployment \
    --region us-east-1
```

## Verifying Environment Variables

Check environment variables in a running container:

```bash
# Get task ID
TASK_ID=$(aws ecs list-tasks \
    --cluster manuales-hogar-ai-cluster \
    --service-name manuales-hogar-ai-service \
    --region us-east-1 \
    --query 'taskArns[0]' \
    --output text | cut -d'/' -f3)

# Execute command in container
aws ecs execute-command \
    --cluster manuales-hogar-ai-cluster \
    --task $TASK_ID \
    --container manuales-hogar-ai-container \
    --command "env" \
    --interactive \
    --region us-east-1
```

## Troubleshooting

### Secret Not Found

If you get "Secret not found" errors:

1. Verify secret exists:
```bash
aws secretsmanager describe-secret \
    --secret-id manuales-hogar-ai/openrouter-api-key \
    --region us-east-1
```

2. Check IAM permissions for ECS task role
3. Verify secret ARN in task definition

### Environment Variable Not Applied

1. Restart ECS service:
```bash
aws ecs update-service \
    --cluster manuales-hogar-ai-cluster \
    --service manuales-hogar-ai-service \
    --force-new-deployment \
    --region us-east-1
```

2. Check CloudWatch logs for errors
3. Verify task definition was updated
