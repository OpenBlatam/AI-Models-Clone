## 🗂️ Estructura de Tests

### Tests Unitarios
- `components/` - Tests de componentes React
- `lib/hooks/` - Tests de hooks personalizados
- `lib/store/` - Tests de Zustand store
- `lib/api/` - Tests de servicios API
- `lib/utils/` - Tests de utilidades
- `lib/validations/` - Tests de validaciones
- `lib/constants/` - Tests de constantes
- `lib/config/` - Tests de configuración
- `lib/types/` - Tests de tipos

### Tests E2E
- `e2e/user-flows.test.tsx` - Flujos de usuario
- `e2e/music-workflow.test.tsx` - Workflow de música
- `e2e/accessibility.test.tsx` - Accesibilidad
- `e2e/performance.test.tsx` - Performance
- `e2e/advanced-workflows.test.tsx` - Workflows avanzados
- `e2e/edge-cases.test.tsx` - Casos edge
- `e2e/stress-test.test.tsx` - Tests de estrés
- `e2e/cross-component.test.tsx` - Integración cross-component

### Tests Adicionales
- `integration/` - Tests de integración
- `regression/` - Tests de regresión
- `snapshots/` - Snapshot tests
- `performance/` - Tests de performance

### Utilities
- `setup/test-utils.tsx` - Test utilities centralizadas
- `setup/test-utils.test.ts` - Tests de utilities
- `helpers/test-helpers.test.ts` - Test helpers

# Todos los tests

# Modo watch
npm run test:watch

# Solo E2E
npm test -- e2e
